Sending an immediate response

If you have finished Hello World example, well done. You have already sent an immediate response back to Slack. As you can see, the default return in process() function is an immediate type message.

Slack has provided us with a message structure that allows us to send back more styled message format.

From Slack:

You can include more substantive info in the body of your HTTP 200 response. In fact, you can use any of the complex formatting or attachment fields that are available when sending any message.

Don't worry about what the formatting or attachment fields are, the Slack Framework will cover them nicely in Slack Message section.

Let's try to send a more advanced message.

Open any Command class that you have implemented before and include the following code in process() function.

The message structure is specified by Slack Official guide.

    ...
    public function process()
    {
        return [
            "text" => "Hey, Slack Framework!",
            "attachments" => [
                [
                    "text" => "Hey, Friend :)"
                ]
            ]
        ];
    }
    ...

Limitation

  • The message must be received by Slack within 3000 milliseconds of the original request being sent, otherwise a Timeout was reached be displayed to the user.

Using Slack Framework to create message

The above array message can be nicely transformed into the following PHP data-driven structure:

return SlackMessage::create()
            ->setText('Hey, Slack Framework!')
            ->withAttachment(
                SlackAttachment::create()
                    ->setText('Hey, Friend :)')
            )->toArray();

Please read Slack Message section for more details.

A little introduction about Slack Message

Slack has two types of message structure:

  • Slack Message (outmoded)

Reference: https://api.slack.com/docs/message-attachments and https://api.slack.com/docs/message-formatting

  • Block Kit (new mode)

Reference: https://api.slack.com/messaging
